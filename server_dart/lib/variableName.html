  • ğŸ‘ time // this is utc/gmt time, which is also gps_time What is GPS TIme

That Quora answer is factually wrong. GPS time is off of UTC (GMT is obsolete) by several seconds. See, e.g., https://en.wikipedia.org/wiki/Global_Positioning_System#Timekeeping

The GPS navigation message includes the difference between GPS time and UTC. As of January 2017, GPS time is 18 seconds ahead of UTC because of the leap second added to UTC on December 31, 2016.[148] Receivers subtract this offset from GPS time to calculate UTC and specific time zone values. New GPS units may not show the correct UTC time until after receiving the UTC offset message. The GPS-UTC offset field can accommodate 255 leap seconds (eight bits).

It's not that we can't easily convert GPS time to UTC time, it's just that we have to remember to do it :)

unix_time, - don't use, use utc. date -u or it's programatic equivelant will provide utc on unix UTC on Linux

Internally, all Unix time operations are in unix time; UTC is essentially just an output format. Sometimes we can't hide from the UNIX time, esp. when other APIs send it to us for whatever reason. It's worth converting to whatever programmatic time representation is available in the host programming language when we can, and exchanging as RFC3339, as suggested in the proposed Glossary and conventions

gps_time, - Delete as duplicate of time

This is wrong, as above.

iridium_time - according to the Documentation iridium timestamp as received by the webhook are utc already.

They are at least putting a UTC suffix on the webhook timestamps -- I'd be interested in seeing if it's accurate. Regardless,
that doesn't help the Tensor which is dealing with this mush - https://docs.rockblock.rock7.com/docs/whats-the-time

This number corresponds to the number of 90-millisecond frames elapsed since the new Iridium Time Era (ERA2) which superceded ERA1. The start of ERA2 is defined as May 11, 2014, at 14:23:55 UTC/GMT, while the start of ERA1 was March 8, 2007, 03:50:21.00 UTC/GMT. Please note that both ERA1 and ERA2 are different epoch times from UNIX epoch time, which is defined as the number of seconds elapsed since Jan 1, 1970 UTC/GMT.
